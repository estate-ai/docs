# 6. 시스템 아키텍처

![시스템 아키텍처 구성도](assets/archtect/ChatGPT%20Image%202026%EB%85%84%206%EC%9B%94%2017%EC%9D%BC%20%EC%98%A4%ED%9B%84%2001_55_38.png)

## 1. 아키텍처 개요

* 본 프로젝트는 Next.js 기반 웹 클라이언트, API 게이트웨이, 인증 서버, 백엔드 서비스, AI/ML 서비스, 데이터 저장소, 외부 API 연동으로 구성된다.
* 전체 서비스는 VPS 내부의 컨테이너 네트워크에서 실행된다.
* 프론트엔드 요청은 CDN/Frontend 영역을 거쳐 API 게이트웨이로 전달된다.
* API 게이트웨이는 각 백엔드 서비스와 AI/ML 서비스로 요청을 라우팅한다.
* 인증은 별도의 인증 서버에서 처리하며, 백엔드 서비스는 JWT를 검증하여 사용자별 데이터 접근을 제어한다.
* MVP의 기본 분석 단위는 서울 행정동이다.

## 2. 전체 구성

### 2.1 Frontend / CDN

* Next.js 웹 클라이언트
* Next.js 정적 자산
* SSR / RSC 서버 렌더링
* Edge Functions 또는 프론트엔드 프록시
* 주요 화면

  * 메인 페이지
  * 온보딩 페이지
  * 지도 기반 상권 탐색 페이지
  * 상권 상세 페이지
  * AI 리포트 생성 페이지
  * 개인 페이지

### 2.2 Gateway / Auth

* API 게이트웨이

  * CORS / OPTIONS 처리
  * 경로 기반 라우팅
  * Authorization 헤더 전달
  * SSE / Streaming pass-through
  * Docker Compose Label 기반 서비스 라우팅
* 인증 서버

  * OAuth 2.0 / OpenID Connect 기반 로그인
  * Google, Kakao, Naver OAuth 연동
  * JWT 발급
  * 통합 인증 처리

### 2.3 Backend Services

* 사용자 / 프로필 서비스

  * 사용자 정보 관리
  * 온보딩 응답 저장
  * 개인 취향 분석 결과 저장
  * 개인 페이지 데이터 조회
* 상권 데이터 서비스

  * 서울 행정동 기반 상권 데이터 조회
  * 상권 프로필 제공
  * 지도 화면용 상권/행정동 데이터 제공
* 프랜차이즈 서비스

  * 프랜차이즈 시드 데이터 관리
  * 브랜드별 매출, 창업비용, 업종, 소개 정보 제공
  * 프랜차이즈 목록 및 상세 제공
* 추천 / 점수 산정 서비스

  * 사용자 조건 분석
  * 상권-업종 적합도 계산
  * 브랜드 점수 계산
  * 상권 + 업종 + 브랜드 추천 조합 생성
* 리포트 / 문서 서비스

  * AI 리포트 생성 요청 관리
  * 생성 리포트 저장
  * 상권 리포트, 상권 기획, 사업 생애주기 문서 관리

### 2.4 AI / ML Services

* AI 에이전트 서버

  * AI 채팅
  * SSE 기반 스트리밍 응답
  * Tool Call Loop
  * HITL 승인 흐름
  * 리포트 생성을 위한 대화 세션 관리
* RAG 검색 모듈

  * 공공데이터, 상권 데이터, 프랜차이즈 정보 검색
  * Top-K 검색
  * 반복 검색 루프
  * AI 리포트 생성 근거 제공
* 추천 / 예측 모델

  * 사용자 선호도 벡터 생성
  * 행정동/상권 벡터 생성
  * 업종/프랜차이즈 벡터 생성
  * 추천 후보 산출
* Evals / Benchmark 모듈

  * AI 에이전트 응답 품질 평가
  * RAG 검색 정확도 평가
  * Tool Call 성공 여부 평가
  * 리포트 생성 품질 평가

### 2.5 Data Layer

* PostgreSQL

  * 사용자, 온보딩, 프랜차이즈, 추천 결과, 리포트 메타데이터 저장
* PostGIS

  * 행정동 경계, 점포 위치, 집객시설 위치 등 공간 데이터 저장
* PGVector

  * RAG 임베딩 저장
  * 사용자 선호도 벡터 저장
  * 행정동/업종/프랜차이즈 벡터 저장
* Redis

  * 캐시 및 임시 상태 저장
* RabbitMQ

  * 리포트 생성, 데이터 처리, AI 작업 등 비동기 작업 처리
* Object Storage

  * 생성 리포트 파일
  * 문서 파일
  * 원천 데이터 파일

### 2.6 External APIs

* LLM API
* 지도 API
* Google / Kakao / Naver OAuth Provider
* 서울시 상권분석서비스
* 서울 열린데이터광장
* 공정거래위원회 프랜차이즈 정보공개서
* 공공데이터포털 프랜차이즈 API

## 3. 주요 요청 흐름

### 3.1 로그인 흐름

* 사용자 → Next.js 웹 클라이언트
* Next.js 웹 클라이언트 → 인증 서버
* 인증 서버 → 외부 OAuth Provider
* 인증 서버 → JWT 발급
* 프론트엔드 → API 게이트웨이 → 백엔드 서비스
* 백엔드 서비스에서 JWT 검증

### 3.2 온보딩 기반 추천 흐름

* 사용자 → 온보딩 페이지
* 온보딩 응답 → 사용자 / 프로필 서비스
* 사용자 조건 → 추천 / 점수 산정 서비스
* 추천 / 점수 산정 서비스 → 상권 데이터 서비스
* 추천 / 점수 산정 서비스 → 프랜차이즈 서비스
* 추천 / 점수 산정 서비스 → 추천 / 예측 모델
* 결과 → 상권 + 업종 + 브랜드 추천 조합 반환

### 3.3 지도 기반 상권 검색 흐름

* 사용자 → 지도 기반 상권 탐색 페이지
* 검색 조건 → API 게이트웨이
* API 게이트웨이 → 상권 데이터 서비스
* 상권 데이터 서비스 → PostGIS
* 결과 → 지도 화면에 추천 행정동/상권 후보 표시

### 3.4 AI 리포트 생성 흐름

* 사용자 → AI 리포트 생성 페이지
* 사용자 질문 또는 리포트 생성 요청 → AI 에이전트 서버
* AI 에이전트 서버 → RAG 검색 모듈
* RAG 검색 모듈 → PGVector / PostgreSQL
* AI 에이전트 서버 → LLM API
* 생성 결과 → 리포트 / 문서 서비스
* 리포트 / 문서 서비스 → PostgreSQL / Object Storage
* 결과 → 개인 페이지에서 조회

### 3.5 개인 페이지 조회 흐름

* 사용자 → 개인 페이지
* 요청 → 사용자 / 프로필 서비스
* 요청 → 리포트 / 문서 서비스
* 서비스 → PostgreSQL / Object Storage
* 결과 → 저장 리포트, 창업 문서, 개인 취향 분석 결과 표시

## 4. 설계 원칙

* 추천 점수와 LLM 설명 생성을 분리한다.
* 추천 점수는 공공데이터, 상권 데이터, 업종 데이터, 프랜차이즈 데이터, 사용자 조건을 기반으로 산정한다.
* LLM은 계산된 점수와 RAG 검색 근거를 바탕으로 추천 이유, 창업 리스크, 운영 전략, AI 리포트를 생성한다. 
* LLM은 사용자 성향을 제외한 DB는 직접 조작하지 않으며, 조회는 사용자 권한을 이양받아 조회한다.
* MVP 기본 분석 단위는 서울 행정동으로 설정한다.
* 시, 자치구, 상권, 좌표 단위 데이터는 행정동 중심으로 매핑한다.
* API 게이트웨이를 통해 백엔드 서비스와 AI/ML 서비스를 분리한다.
* AI 에이전트는 HITL, Tool Call Loop, Evals를 포함한 하네스 구조로 설계한다.
