# 4. MVP 범위

## 1. 해결할 문제 정의

### MVP가 해결하려는 문제

예비 창업자는 프랜차이즈 정보, 상권 데이터, 창업지원 정보를 각각 다른 서비스에서 따로 확인해야 한다.
특히 초기 창업자는 상권 데이터의 매출, 유동인구, 소비층, 점포 수, 임대료, 생존률 등의 지표를 자신의 예산, 업종 선호, 운영 방식, 위험 선호도와 연결해 해석하기 어렵다.

기존 서비스는 프랜차이즈 정보 제공 또는 상권분석 정보 제공에는 강점이 있으나, 사용자의 창업 조건을 기준으로 **상권 × 업종 × 프랜차이즈 조합**을 추천하고, 추천 이유와 리스크를 설명하는 통합 흐름은 부족하다.

### 사용자 니즈

* 사용자는 창업 지식이 부족해도 자신의 조건에 맞는 창업 후보를 추천받고 싶다.
* 사용자는 예산, 지역, 목표 수익, 운영 가능 시간, 희망 업종, 창업 규모, 선호 브랜드, 위험 선호도 등을 반영한 추천을 받고 싶다.
* 사용자는 단순한 상권 데이터가 아니라 “왜 이 상권과 업종이 나에게 맞는지”를 이해하고 싶다.
* 사용자는 추천 결과를 지도, 상권 상세, 리포트, 개인 페이지로 이어서 확인하고 관리하고 싶다.

### 가치 제안

본 MVP는 서울 행정동 단위 상권 데이터와 프랜차이즈 데이터를 기반으로, 예비 창업자에게 **상권 + 업종 + 브랜드 조합**을 추천한다.

추천 결과는 단순 점수로 끝나지 않고, 추천 이유, 주요 근거 데이터, 창업 리스크, 운영 전략을 함께 제공한다.
LLM은 추천 점수를 직접 생성하지 않고, 공공데이터 기반 추천 점수와 RAG 검색 근거를 바탕으로 사용자가 이해하기 쉬운 설명과 리포트를 생성하는 역할을 맡는다.

## 2. 경쟁 환경 / 기존 대안

### 사용자가 현재 문제를 해결하는 방식

* 프랜차이즈 브랜드 정보는 마이프차 같은 프랜차이즈 플랫폼에서 확인한다.
* 상권 데이터는 서울시 상권분석서비스 또는 소상공인365 같은 공공 상권분석 서비스를 사용한다.
* 창업지원사업, 교육, 창업시설 정보는 스타트업플러스 같은 창업지원 플랫폼에서 확인한다.
* 결과적으로 사용자는 여러 서비스를 오가며 직접 정보를 수집하고, 스스로 비교·해석해야 한다.

### 기존 서비스 또는 대안

| 서비스         | 주요 역할                            | 강점                              | 한계                                   |
| ----------- | -------------------------------- | ------------------------------- | ------------------------------------ |
| 마이프차        | 프랜차이즈 브랜드 탐색, 창업 정보, 상권정보, 수익계산기 | 프랜차이즈 브랜드 정보와 창업 콘텐츠가 풍부함       | 상권 데이터와 사용자 조건 기반 추천이 완전히 통합된 구조는 약함 |
| 서울시 상권분석서비스 | 서울 골목상권 빅데이터 분석                  | 매출, 인구, 점포, 생존률 등 서울 상권 데이터가 강함 | 프랜차이즈 브랜드 추천과 사용자 선호 기반 추천은 약함       |
| 소상공인365     | 상권분석, 경영진단, 소상공인 지원 정보           | 공공데이터 기반 경영·상권 분석에 강함           | 예비 창업자의 프랜차이즈 선택 흐름과 직접 연결되지는 않음     |
| 스타트업플러스     | 서울시 통합 창업 플랫폼                    | 창업지원사업, 투자 매칭, 기술거래, 창업시설 정보 제공 | 자영업·프랜차이즈 입지 추천 서비스와는 성격이 다름         |

### 기존 방식의 한계

* 프랜차이즈 탐색과 상권분석이 분리되어 있다.
* 상권 데이터는 많지만 비전문가가 해석하기 어렵다.
* 사용자 조건을 기준으로 상권, 업종, 브랜드를 한 번에 추천하는 흐름이 부족하다.
* AI 또는 챗봇 기능이 있어도 추천 점수, 근거 데이터, 리포트, 개인 페이지까지 이어지는 의사결정 흐름은 약하다.
* 창업 리스크를 추천 결과와 함께 설명하는 구조가 부족하다.

### 우리 MVP가 더 잘해야 하는 부분

* 프랜차이즈 정보와 상권 데이터를 분리하지 않고 하나의 추천 조합으로 연결한다.
* 사용자의 예산, 지역, 업종, 운영 가능 시간, 위험 선호도 등을 추천에 반영한다.
* 추천 결과를 **상권 + 업종 + 브랜드 TOP 3** 형태로 제공한다.
* 추천 이유, 근거 데이터, 리스크, 운영 전략을 함께 설명한다.
* 지도, 상권 상세, AI 리포트, 개인 페이지까지 이어지는 최소 사용자 흐름을 제공한다.
* RAG와 LLM을 활용하되, LLM은 설명과 리포트 생성을 담당하고 추천 점수는 데이터 기반 로직으로 산정한다.

## 3. MVP 검증 계획

### 검증할 핵심 가설

1. 사용자는 창업 조건을 입력하면 자신에게 맞는 상권·업종·프랜차이즈 추천 결과를 이해하고 활용할 수 있다.
2. 서울 행정동 단위 상권 데이터는 창업 입지 추천의 기본 분석 단위로 활용 가능하다.
3. 업종 데이터는 상권과 프랜차이즈를 연결하는 핵심 중간 기준으로 작동할 수 있다.
4. 공공데이터 기반 점수와 RAG 기반 설명을 결합하면, 사용자는 추천 결과의 이유와 리스크를 더 쉽게 이해할 수 있다.
5. AI 리포트는 추천 결과를 창업 의사결정 문서로 전환하는 데 도움이 된다.

### 테스트 대상 사용자

* 창업 지식이 부족한 초기 예비 창업자
* 예산과 관심 업종이 있는 실제 창업 준비자
* 프랜차이즈 창업을 고려하는 사용자
* 상권과 리포트를 저장하고 비교하려는 사용자

### 테스트할 핵심 기능

* 소셜 로그인
* 유저 온보딩 설문
* 프랜차이즈 시드 데이터 및 브랜드 게시글
* 서울 행정동 기반 상권 조회/검색
* 지도 기반 상권 시각화
* 상권 프로필 생성
* 지역 × 업종 적합도 산정
* 추천 브랜드 TOP 3 제공
* 추천 이유 및 리스크 설명
* RAG 기반 데이터 검색
* AI 챗봇 질문 기능
* 상권 리포트 생성
* 상권 키워드 그래프 시각화
* 개인 페이지 내 리포트 및 창업 문서 관리

### 검증 기준

* 사용자가 온보딩을 완료하고 추천 결과까지 도달할 수 있다.
* 사용자가 추천 결과에서 상권, 업종, 브랜드, 추천 이유, 리스크를 확인할 수 있다.
* 사용자가 지도에서 추천 행정동 또는 상권 후보를 확인할 수 있다.
* 사용자가 특정 상권 또는 추천 결과를 기반으로 AI 리포트를 생성할 수 있다.
* 생성된 리포트가 개인 페이지에 저장되고 다시 조회될 수 있다.
* 추천 점수와 LLM 설명의 역할이 분리되어 있으며, LLM이 임의로 점수를 생성하지 않는다.
* 최소 1개 이상의 추천/예측 모델 또는 점수 산정 로직이 서비스 흐름에 연결된다.
