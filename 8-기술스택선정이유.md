# 8. 기술스택 선정근거

## 1. Frontend

### Next.js

* 라우트 기반 페이지 렌더링 및 번들 사이즈 성능 최적화
* 프론트엔드 단의 자체적인 보안을 추가할 수 있으며, 백엔드와 같은 VPS 내에서 SSR 성능 향상 이점이 있다.

### Orval

* 백엔드 API 변경 사항을 프론트엔드 타입과 빠르게 동기화하기 위해 사용하며, 이형서버 조합에서의 tRPC를 대체한다.

### Langgraph/React

* 랭그래프 서버의 SSE 통신을 추상화한다.


---

## 2. Backend

### Java 21

* Spring Boot 기반 백엔드 서비스 개발에 적합한 안정적인 장기 지원 버전이다.
* 인증, 데이터 처리, 추천 API, 문서 관리 API처럼 복잡한 도메인 로직을 안정적으로 구현할 수 있다.

### Spring Boot

* 사용자, 상권, 프랜차이즈, 추천, 리포트 서비스를 도메인 단위로 구성하기 적합하다.
* REST API, 인증 연동, JPA, OpenAPI 문서화와의 통합이 쉽다.

### JPA

* ERD 기반 도메인 모델을 객체 중심으로 구현할 수 있다.
* 사용자, 온보딩, 추천 결과, 리포트, 프랜차이즈 데이터처럼 관계형 데이터가 많은 구조에 적합하다.

### flyway

* SQL을 각 개발 참여자들과 배포 서버에서 동일하게 마이그레이트하기 위해 사용된다.
* SQL을 형상관리하여 변경사항을 추적 및 디버깅할 수 있다.


---

## 3. Gateway / Auth

### Traefik + Docker Compose

* Docker Compose Label 기반으로 서비스 라우팅을 구성할 수 있어 별도 서비스 디스커버리 없이 MSA 구조를 단순하게 운영할 수 있다.
* API 게이트웨이 역할을 맡아 프론트엔드 요청을 각 백엔드 서비스와 AI 서비스로 전달한다.

### Authentik

* OAuth 2.0 / OpenID Connect 기반 인증 서버로 사용하여 소셜 로그인과 JWT 발급을 통합 관리한다.
* 인증 로직을 각 서비스에 분산하지 않고 중앙 인증 서버에서 처리하기 위해 사용한다.

---

## 4. AI / Agent

### LangGraph API

* 상태 기반 AI 에이전트 실행, Tool Call Loop, HITL 흐름을 구현하기 위한 표준 API로서 노출된다.

### RAG

* 공공데이터, 상권 데이터, 프랜차이즈 정보를 검색 근거로 활용하기 위해 사용한다.
* LLM이 임의로 추천 점수를 생성하지 않고, 검색된 근거를 바탕으로 설명과 리포트를 생성하도록 제한한다.

---

## 5. Database / Data

### PostgreSQL

* 사용자, 온보딩, 추천 결과, 프랜차이즈, 리포트 등 핵심 관계형 데이터를 저장하기 위해 사용한다.
* PostGIS와 PGVector를 함께 사용할 수 있어 데이터 저장소 구성을 단순화할 수 있다.

### Redis

* 자주 조회되는 상권 프로필, 추천 결과, 임시 상태를 캐싱하기 위해 사용한다.
* 지도와 추천 API의 반복 조회 성능을 보완할 수 있다.

### RabbitMQ

* 리포트 생성, 데이터 처리, AI 작업처럼 시간이 걸리는 작업을 비동기 처리하기 위해 사용한다.
* 사용자가 요청 후 오래 기다리지 않도록 백그라운드 작업 큐로 분리한다.

### Object Storage (minio)

* S3 호환 API를 제공하는 오픈소스 스토리지 minio를 통해 설정을 통제한다.
* DB에는 메타데이터를 저장하고, 실제 파일은 별도 저장소에 분리한다.

---

## 6. Infra / Deployment

### Docker Compose

* 프론트엔드, 백엔드, AI 서버, DB, Redis, RabbitMQ 등을 로컬과 서버에서 동일한 방식으로 실행하기 위해 사용한다.
* 캡스톤 기간 내 빠른 개발과 배포 환경 재현성을 확보할 수 있다.

---

## 7. External APIs

### 카카오 지도 API

* 서울 행정동과 추천 상권 후보를 지도 위에 시각화하기 위해 사용한다.
* 사용자가 추천 결과를 공간적으로 비교할 수 있도록 한다.

### 서울시 공공데이터 / 서울 열린데이터광장

* 행정동 단위 매출, 소비, 인구, 점포, 집객시설, 상권변화지표 등을 수집하기 위해 사용한다.
* 상권-업종 적합도와 창업 리스크 산정의 핵심 근거 데이터로 활용한다.

### 공정거래위원회 / 공공데이터포털 프랜차이즈 API

* 프랜차이즈 정보공개서, 창업비용, 브랜드별 가맹점 현황 데이터를 수집하기 위해 사용한다.
* 브랜드 점수 산정과 프랜차이즈 추천 근거 데이터로 활용한다.
