## 3. 기능 시연 흐름

### 3.1 온보딩 기반 추천

* 사용자가 로그인한다.
* 온보딩 설문에 답변한다.
* 예산, 희망 지역, 목표 수익, 운영 가능 시간, 희망 업종, 위험 선호도를 입력한다.
* 서비스는 사용자 조건을 바탕으로 추천 후보를 생성한다.
* 추천 결과로 상권 + 업종 + 브랜드 조합을 보여준다.

### 3.2 지도 기반 상권 탐색

* 사용자가 지도 페이지로 이동한다.
* 왼쪽 사이드바에서 상권 검색 조건을 입력한다.
* 지도 위에 추천 행정동 또는 상권 후보가 표시된다.
* 특정 상권을 선택하면 2단 사이드바가 열린다.
* 상단에는 상권 리포트 요약과 그래프가 표시된다.
* 하단에는 추천 프랜차이즈 리스트가 표시된다.

### 3.3 AI 리포트 생성

* 사용자가 추천 상권 또는 프랜차이즈를 선택한다.
* AI 리포트 생성 페이지로 이동한다.
* AI 에이전트와 대화하며 질문을 구체화한다.
* AI는 RAG 검색 결과와 추천 점수 근거를 바탕으로 리포트를 생성한다.
* 생성된 리포트는 개인 페이지에 저장된다.

## 4. 기술 발표 핵심 메시지

### 4.1 머신러닝 추천

* 추천은 단순 필터링이 아니라 사용자 조건, 행정동 데이터, 업종/프랜차이즈 데이터를 함께 활용한다.
* 사용자 타워, 행정동 타워, 업종/프랜차이즈 타워를 통해 추천 후보를 산출한다.
* MVP에서는 벡터 유사도 또는 점수 산정 로직을 활용하고, 향후 학습 기반 투타워 모델로 확장한다.

### 4.2 AI 자율형 에이전트

* AI 에이전트는 LangGraph 기반 Tool Call Loop로 동작한다.
* AI 응답에 ToolCalls가 있으면 도구 실행 흐름으로 이동한다.
* ToolCalls가 비어 있으면 최종 응답으로 진행한다.
* 도구 호출 시 사용자 권한이나 정책에 따라 인터럽트를 발생시킨다.
* 인터럽트는 LangGraph React를 통해 클라이언트에 표시된다.
* 사용자가 승인하거나 답변하면 실행이 재개된다.

### 4.3 추천 점수와 LLM 역할 분리

* LLM은 추천 점수를 직접 생성하지 않는다.
* 추천 점수는 데이터 기반 로직 또는 모델이 산정한다.
* LLM은 계산된 점수와 RAG 검색 근거를 바탕으로 추천 이유, 리스크, 운영 전략, 리포트를 생성한다.

## 7. 발표 마무리 메시지

본 프로젝트는 단순한 상권 정보 조회 서비스가 아니라, 예비 창업자의 조건을 기준으로 **상권 × 업종 × 프랜차이즈 조합**을 추천하는 AI 기반 창업 의사결정 플랫폼이다.

핵심 기술은 서울 행정동 단위 공공데이터를 활용한 추천/예측 모델과, RAG 및 Tool Call Loop를 갖춘 AI 자율형 에이전트이다.

MVP에서는 예비 창업자가 창업 조건을 입력하고, 추천 상권과 프랜차이즈를 확인한 뒤, AI 리포트로 의사결정을 이어가는 최소 흐름을 구현한다.
